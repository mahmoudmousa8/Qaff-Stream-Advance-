# قائمة المهام (Task List)

## 1. فك ارتباط موعد البدء وموعد الإيقاف (Decouple Start/Stop Times)
- [ ] استبدال حقل المدة (ساعات ودقائق منسدلة) في جدول القنوات بمدخل نصي للتاريخ المباشر `MM-DD HH:MM` ومعه `DateTimePicker` لتعيين موعد الإيقاف.
  - [ ] تحديث العرض الأول (Table view) في `src/app/page.tsx`.
  - [ ] تحديث العرض الثاني (Card view / Mobile view) في `src/app/page.tsx`.
  - [ ] إلغاء أي دوال تحديث أو عمليات ربط تقوم بتعديل وقت الإيقاف تلقائياً عند تغيير وقت البدء يدوياً.

## 2. أزرار أوقات الإيقاف والجدولة السريعة الفردية
- [ ] وضع حاوية أزرار سريعة بعد حقل موعد الإيقاف مباشرة في كلا العرضين (جدول وبطاقات) تحتوي على:
  - [ ] زر "أقرب 20": يجدول لأقرب 20 دقيقة، إيقاف بعد 13 دقيقة.
  - [ ] زر "أقرب 30": يجدول لأقرب 30 دقيقة، إيقاف بعد 24 دقيقة.
  - [ ] زر "أقرب ساعة": يجدول لأقرب ساعة، إيقاف بعد 50 دقيقة.
  - [ ] زر "أقرب ساعتين": يجدول لأقرب ساعتين، إيقاف بعد ساعة و50 دقيقة.
- [ ] كتابة دوال المعالجة الفردية للجدولة السريعة (`handleClosest30Schedule`, `handleClosestHourSchedule`, `handleClosest2HourSchedule`) على الواجهة.

## 3. أزرار الجدولة الجماعية في البار العلوي (Bulk Actions)
- [ ] إضافة زر الجدولة الجماعية "بث مسجل فقط للكل" (File Only All) لتعيين إدخال ملف مسجل وتعطيل التبديل لكافة القنوات غير النشطة.
- [ ] إضافة زر الجدولة الجماعية "أقرب 30 للكل" (المدى 24 دقيقة).
- [ ] إضافة زر الجدولة الجماعية "أقرب ساعة للكل" (المدى 50 دقيقة).
- [ ] إضافة زر الجدولة الجماعية "أقرب ساعتين للكل" (المدى ساعة و50 دقيقة).
- [ ] تنفيذ المسارات المقابلة في محرك الإجراءات الجماعية بالخلفية `src/app/api/slots/bulk/route.ts`.

## 4. زيادة مسارات الـ Slots بدون حدود في لوحة تحكم الأدمين (Unlimited Slots)
- [ ] تعديل الحدود القصوى في نموذج الأدمين لتصل لـ 5000 أو كتابة أي قيمة رقمية بدون قيود.
- [ ] تحسين مسار جلب القنوات `src/app/api/slots/route.ts` بحيث يقوم بإنشاء القنوات في قاعدة البيانات تدريجياً وبصفة مؤقتة (On-Demand) بناءً على الصفحة النشطة الحالية فقط وليس دفعة واحدة لتفادي تجميد اللوحة.

## 5. واجهة دمج الذكاء الاصطناعي (Google Gemini Chat Mode)
- [ ] إنشاء مسار خلفية جديد لمعالجة المحادثة `src/app/api/ai/chat/route.ts` يتصل بـ Gemini 2.5 Flash باستخدام مفتاح المستخدم المدخل.
- [ ] تصميم نافذة محادثة ذكية وراقية (`Dialog` / `Card`) في الواجهة الرئيسية `src/app/page.tsx` تفتح من خلال زر بأيقونة `Sparkles` في البار العلوي.
- [ ] دعم حفظ مفتاح API وحالات المحادثة في الـ `localStorage` للمتصفح.
- [ ] إضافة ميزة استخراج قوائم العناوين والأوصاف المقترحة من محتوى الدردشة تلقائياً وإتاحة زر "حفظ كقائمة جديدة في اللوحة" للمستخدم.

## 6. الفحص والاختبار والتحقق (Verification & Testing)
- [ ] إجراء فحص شامل عن طريق تشغيل الـ Linter و TypeScript Compiler للتأكد من خلو المشروع من أخطاء البناء.
- [ ] التحقق من استقلالية المدخلات وتوزيع الأوقات والتحميل التدريجي للمسارات بكفاءة.
- [ ] تجربة واجهة Gemini AI والتأكد من نجاح حفظ القوائم المتولدة تلقائياً.
